En esta práctica trabajamos en equipo usando GitHub para crear una página web sencilla con un formulario donde el usuario puede escribir su nombre y recibir un mensaje. Primero, el líder creó el repositorio y subió la estructura básica del archivo index.html. Después se crearon las ramas estilos y logica para dividir el trabajo, de manera que cada integrante pudiera trabajar por separado sin afectar a los demás. Cada uno trabajó en su parte usando comandos como git add, git commit, git push, git branch y git checkout para guardar y subir los cambios. En la rama de estilos se mejoró la apariencia de la página con CSS, como los colores, botones y diseño en general, mientras que en la rama de lógica se programó la funcionalidad con JavaScript para validar si el usuario escribe algo y mostrar un mensaje según el caso. Al final, el líder juntó todo usando merge en la rama principal main, logrando que el proyecto funcionara correctamente. Como resultado, se obtuvo una página funcional, con validación y un diseño agradable. En general, esta práctica ayudó a entender cómo trabajar en equipo usando ramas en GitHub y lo útil que es para organizar mejor el trabajo.
